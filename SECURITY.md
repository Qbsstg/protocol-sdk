# Security Policy

## Supported Versions

Security fixes are currently applied to the latest development branch until the
first stable release is published.

## Reporting a Vulnerability

Please report security issues privately through GitHub Security Advisories when
the public repository is available. If advisories are not enabled yet, contact
the repository owner directly and avoid opening a public issue with exploit
details.

Reports should include:

- affected module and version or commit
- input bytes or frame examples when safe to share
- expected impact
- reproduction steps

Please do not include production credentials, private hostnames, or customer
data in vulnerability reports.
